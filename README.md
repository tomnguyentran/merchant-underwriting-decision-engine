### Technical Summary

* **Objective:** Build an automated underwriting system to process 2,000 merchant applications, prioritizing regulatory compliance and financial risk detection.
* **Tools Used:**
  * **Python:** Engineered a synthetic dataset of 2,000 applications with realistic risk distributions.
  * **SQL:** Constructed the decision logic to categorize applicants.
  * **Excel:** Executive summary reporting of portfolio exposure and decision funnel.
* **Repository Structure:** 
  * `/summary_output`: Executive report visualizing the final assessments.
  * `/data`: Synthetic merchant application datasets.
  * `/scripts`: Source code used for data generation.
  * `/queries`: SQL logic for the underwriting model and reporting.
